# Dossier Analyse et Conception - PFE
# Application: Cercle Bleu Enquetes

## 1) Presentation du projet

Cercle Bleu Enquetes est une application mobile Android permettant la collecte, la centralisation, la gestion et l'export des donnees scientifiques liees au crabe bleu dans le cadre du projet Interreg NEXT Italie-Tunisie.

Elle remplace un mode manuel (papier/fichiers disperses) par un systeme numerique base sur:
- Flutter (application mobile)
- Firebase Authentication
- Cloud Firestore
- Firebase Storage
- Services d'export CSV/PDF

Le systeme couvre:
- Chaine Terrain (observation, captures, variables environnementales, remarques, medias)
- Chaine Laboratoire (analyses biologiques, epibiontes, rattachement observation)
- Supervision admin (gestion chercheurs, gestion enquetes, statistiques, dossier medias)

## 2) Problematique

Le processus actuel de collecte souffre de:
- pertes de donnees
- incoherences de saisie
- absence de centralisation
- faible tracabilite
- difficulte d'analyse scientifique
- acces tardif des admins aux informations

## 3) Objectifs

1. Numeriser la collecte terrain et laboratoire.
2. Fiabiliser les donnees par validation et autosave.
3. Centraliser les informations dans Firestore.
4. Ajouter des preuves multimedia (photos, audios).
5. Faciliter l'exploitation via export CSV/PDF.
6. Offrir une gouvernance admin complete (filtres, stats, supervision).

## 4) Acteurs

### Administrateur
- gere les utilisateurs chercheurs
- consulte toutes les enquetes
- filtre et exporte les donnees
- consulte le dossier global photos/audio

### Chercheur / Enqueteur / Analyste labo
- cree et complete les formulaires terrain/labo
- ajoute des pieces jointes
- exporte ses formulaires

### Systeme Firebase
- authentifie les utilisateurs
- persiste les formulaires et metadonnees
- stocke les fichiers
- applique les regles de securite

## 6) Besoins non fonctionnels

- Performance: ecrans reactifs sur smartphone Android
- Disponibilite: reprise apres micro-coupures
- Securite: controle d'acces par role/ownership
- Confidentialite: isolation des donnees utilisateurs
- Integrite: validation champs et coherence metier
- UX/UI: interface lisible et simple sur terrain
- Maintenabilite: architecture modulaire services/ecrans/modeles
- Scalabilite: support augmentation des formulaires et medias
- Auditabilite: createdAt, updatedAt, createdBy

## 7) Regles de gestion

1. Chaque formulaire possede un ownerId.
2. Le chercheur agit sur ses formulaires (admin excepte).
3. Chaque piece jointe reference formId et formType.
4. createdAt et updatedAt assurent la tracabilite.
5. Les champs conditionnels deviennent obligatoires si "Autre".
6. Les filtres n'alterent pas les donnees, seulement la vue.

## 23) Section securite

Mecanismes:
- Authentification Firebase obligatoire.
- Gestion role admin/chercheur.
- Regles Firestore et Storage.
- Validation des entrees.
- Protection fichiers (metadonnees + chemins controles).
- Traçabilite (createdAt, createdBy, ownerId).

Risques / contre-mesures:
- Acces non autorise -> regles role/owner.
- Exposition fichiers -> regles Storage.
- Incoherence donnees -> validations + contraintes metier.
- Reseau instable -> autosave + reprise.
- Echec upload -> messages utilisateur + fallback.

## 24) Conclusion de la conception

La conception de Cercle Bleu Enquetes offre une architecture mobile-cloud robuste, orientee qualite scientifique et tracabilite, avec une UX adaptee au terrain et une gouvernance admin complete. Le dossier est pret a etre integre dans un rapport PFE (analyse et conception UML).
